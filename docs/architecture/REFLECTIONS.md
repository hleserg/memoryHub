# Reflections Schema

This document describes the PostgreSQL schema for storing reflection content in Atman's Reflection Engine.

## Overview

The `public.reflections` table provides persistent storage for reflection content generated by LLMs across all three reflection levels: micro, daily, and deep. This schema is part of Epic E27 (Core Reflections Schema) and enables:

- Persistent storage of reflection content from `OllamaReflectionModel`
- Foreign key relationships for eval.reflection_quality (planned in E0.3)
- Queryable history of agent self-reflection
- Foundation for future embedding-based semantic search (E25)

## Schema

### Table: `public.reflections`

```sql
CREATE TABLE public.reflections (
    id BIGSERIAL PRIMARY KEY,
    agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    level reflection_level NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    session_id UUID REFERENCES sessions(id),
    period_start TIMESTAMPTZ,
    period_end TIMESTAMPTZ,
    content TEXT NOT NULL,
    summary TEXT,
    experience_refs UUID[] NOT NULL DEFAULT '{}',
    reframing_note_ids UUID[] NOT NULL DEFAULT '{}',
    model_provider TEXT,
    model_name TEXT,
    schema_version INTEGER NOT NULL DEFAULT 1,
    metadata JSONB NOT NULL DEFAULT '{}'
);
```

### Columns

| Column | Type | Description |
|--------|------|-------------|
| `id` | BIGSERIAL | Primary key (auto-incrementing integer) |
| `agent_id` | UUID | Agent reference, foreign key to agents(id) |
| `level` | reflection_level | Reflection level: 'micro', 'daily', or 'deep' |
| `created_at` | TIMESTAMPTZ | When this reflection was created |
| `session_id` | UUID | Session reference for micro reflections, NULL for daily/deep |
| `period_start` | TIMESTAMPTZ | Time window start for daily/deep reflections, NULL for micro |
| `period_end` | TIMESTAMPTZ | Time window end for daily/deep reflections, NULL for micro |
| `content` | TEXT | Main reflection content generated by LLM |
| `summary` | TEXT | Optional short title/summary |
| `experience_refs` | UUID[] | Array of experience IDs analyzed in this reflection |
| `reframing_note_ids` | UUID[] | Array of reframing note IDs produced by this reflection |
| `model_provider` | TEXT | LLM provider (e.g., 'ollama', 'anthropic') |
| `model_name` | TEXT | Model name (e.g., 'qwen3:14b') |
| `schema_version` | INTEGER | Schema version for migrations (currently 1) |
| `metadata` | JSONB | Additional metadata as JSONB object |

### Enum: `reflection_level`

```sql
CREATE TYPE reflection_level AS ENUM ('micro', 'daily', 'deep');
```

## Reflection Levels

### Micro Reflections

Micro reflections occur after each session to provide continuity and update the recent narrative layer.

**Characteristics:**
- `level = 'micro'`
- `session_id` is populated (references the session being reflected on)
- `period_start` and `period_end` are NULL
- Typically analyze experiences from a single session
- Focus on immediate learning and narrative updates

**Use case:** After-session summary, recent narrative layer update, immediate reframing

**Example query:**
```sql
SELECT * FROM public.reflections
WHERE level = 'micro'
  AND session_id = '123e4567-e89b-12d3-a456-426614174000'
ORDER BY created_at DESC;
```

### Daily Reflections

Daily reflections look for patterns across multiple sessions and detect recurring themes.

**Characteristics:**
- `level = 'daily'`
- `session_id` is NULL
- `period_start` and `period_end` define the time window (e.g., last 24 hours)
- Analyze experiences across multiple sessions
- Focus on pattern detection and habit formation

**Use case:** Pattern detection, habit identification, value reinforcement

**Example query:**
```sql
SELECT * FROM public.reflections
WHERE level = 'daily'
  AND agent_id = '123e4567-e89b-12d3-a456-426614174000'
  AND period_start >= NOW() - INTERVAL '7 days'
ORDER BY created_at DESC;
```

### Deep Reflections

Deep reflections occur weekly (or on schedule) for comprehensive narrative integration and health assessment.

**Characteristics:**
- `level = 'deep'`
- `session_id` is NULL
- `period_start` and `period_end` define a broader time window (e.g., last 7 days)
- Comprehensive analysis of identity, narrative, and psychological health
- May include Jahoda health criterion assessments

**Use case:** Narrative integration, health assessment, identity evolution tracking

**Example query:**
```sql
SELECT * FROM public.reflections
WHERE level = 'deep'
  AND agent_id = '123e4567-e89b-12d3-a456-426614174000'
ORDER BY created_at DESC
LIMIT 10;
```

## Indexes

The table includes the following indexes for efficient querying:

```sql
-- Timeline queries: get recent reflections for an agent
CREATE INDEX idx_reflections_agent_created
    ON public.reflections(agent_id, created_at DESC);

-- Level-filtered queries: get reflections by level
CREATE INDEX idx_reflections_level_created
    ON public.reflections(level, created_at);

-- Experience back-references: find all reflections that analyzed an experience
CREATE INDEX idx_reflections_experience_refs
    ON public.reflections USING GIN(experience_refs);
```

## Row Level Security (RLS)

The table uses Row Level Security to ensure agents can only access their own reflections:

```sql
ALTER TABLE public.reflections ENABLE ROW LEVEL SECURITY;

CREATE POLICY reflections_isolation ON public.reflections
    USING (agent_id = NULLIF(current_setting('atman.current_agent', TRUE), '')::UUID);
```

To set the agent context:
```sql
SET LOCAL atman.current_agent = '123e4567-e89b-12d3-a456-426614174000';
```

## Python API

### Persistence Model

The persistence layer is in `src/atman/reflection/`:

```python
from atman.reflection.models import ReflectionEvent, ReflectionLevel
from atman.reflection.store import ReflectionStore

# Create a reflection event
event = ReflectionEvent(
    agent_id=agent_uuid,
    level=ReflectionLevel.DAILY,
    content="I noticed a pattern in my recent interactions...",
    summary="Pattern of over-explaining",
    experience_refs=[exp_id1, exp_id2],
    model_provider="ollama",
    model_name="qwen3:14b",
)

# Store it
with ReflectionStore() as store:
    stored = store.add(event)
    print(f"Stored reflection with ID: {stored.id}")
```

### ReflectionStore Methods

```python
class ReflectionStore:
    """PostgreSQL-based storage for reflection content."""

    def add(self, event: ReflectionEvent) -> ReflectionEvent:
        """Add a reflection event to storage."""

    def get(self, reflection_id: int) -> ReflectionEvent | None:
        """Get a reflection by ID."""

    def list_by_session(self, session_id: UUID) -> list[ReflectionEvent]:
        """List all reflections for a specific session."""

    def list_recent(self, agent_id: UUID, limit: int = 10) -> list[ReflectionEvent]:
        """List recent reflections for an agent."""

    def list_by_level(
        self, agent_id: UUID, level: ReflectionLevel, since: datetime | None = None
    ) -> list[ReflectionEvent]:
        """List reflections for an agent at a specific level."""
```

### Automatic Persistence

The `OllamaReflectionModelWithPersistence` adapter automatically persists reflections:

```python
from atman.adapters.reflection import OllamaReflectionModelWithPersistence

with OllamaReflectionModelWithPersistence() as model:
    output = model.generate_reframing_note(experience, context)
    # Reflection is automatically persisted to database
```

## Query Patterns

### Get all reflections for an agent

```sql
SELECT id, level, created_at, summary, content
FROM public.reflections
WHERE agent_id = $1
ORDER BY created_at DESC;
```

### Get reflections for a specific session

```sql
SELECT id, level, created_at, content
FROM public.reflections
WHERE session_id = $1
ORDER BY created_at DESC;
```

### Find reflections that analyzed a specific experience

```sql
SELECT id, level, created_at, summary, content
FROM public.reflections
WHERE $1 = ANY(experience_refs)
ORDER BY created_at DESC;
```

### Get recent reflections by level

```sql
SELECT id, created_at, summary, content
FROM public.reflections
WHERE agent_id = $1 AND level = $2
ORDER BY created_at DESC
LIMIT $3;
```

### Time-series analysis of reflection frequency

```sql
SELECT
    DATE_TRUNC('day', created_at) as day,
    level,
    COUNT(*) as reflection_count
FROM public.reflections
WHERE agent_id = $1
  AND created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE_TRUNC('day', created_at), level
ORDER BY day DESC, level;
```

## Future Extensions

### Embedding Support (E25)

When the EmbeddingPort lands (Epic E25), an embedding column will be added via migration:

```sql
-- Future migration (E25 follow-up)
ALTER TABLE public.reflections
ADD COLUMN embedding VECTOR(768);

CREATE INDEX idx_reflections_embedding
    ON public.reflections USING hnsw(embedding vector_cosine_ops);

COMMENT ON COLUMN public.reflections.embedding IS
    'Vector embedding of reflection content for semantic search';
```

This will enable:
- Semantic similarity search between reflections
- Finding related reflections across different time periods
- Clustering reflections by thematic content
- Vector-based recommendations for reflection review

### Schema Versioning

The `schema_version` column enables safe schema evolution:

- Current version: 1
- Future migrations will increment this value
- Applications can check compatibility before processing

Example migration for version 2:
```sql
ALTER TABLE public.reflections
ADD COLUMN new_field TEXT;

UPDATE public.reflections
SET schema_version = 2
WHERE schema_version = 1;
```

## Troubleshooting

### psycopg not installed

If you see `ImportError: psycopg is required for ReflectionStore`, install it:

```bash
pip install psycopg[binary]
```

Or use the optional dependency:

```bash
pip install "atman[postgres]"
```

### RLS policy blocking access

If queries return no results, check that the agent context is set:

```python
import os
os.environ["ATMAN_CURRENT_AGENT"] = str(agent_uuid)
```

Or in SQL:

```sql
SET LOCAL atman.current_agent = 'your-agent-uuid';
```

### Connection errors

Ensure the database URL is configured:

```bash
export ATMAN_DB_URL="postgresql://atman@localhost:5432/atman"
```
