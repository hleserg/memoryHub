# Reflections Schema

This document describes the PostgreSQL schema for storing reflection content in Atman's Reflection Engine.

## Overview

The `public.reflections` table provides persistent storage for reflection content generated by LLMs across all three reflection levels: micro, daily, and deep. This schema is part of Epic E27 (Core Reflections Schema) and enables:

- Persistent storage of reflection content from `OllamaReflectionModel`
- Foreign key relationships for eval.reflection_quality (planned in E0.3)
- Queryable history of agent self-reflection
- Foundation for future embedding-based semantic search (E25)

## Schema

### Table: `public.reflections`

```sql
CREATE TABLE public.reflections (
    id BIGSERIAL PRIMARY KEY,
    agent_id UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    level reflection_level NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    session_id UUID REFERENCES sessions(id),
    period_start TIMESTAMPTZ,
    period_end TIMESTAMPTZ,
    content TEXT NOT NULL,
    summary TEXT,
    experience_refs UUID[] NOT NULL DEFAULT '{}',
    reframing_note_ids UUID[] NOT NULL DEFAULT '{}',
    model_provider TEXT,
    model_name TEXT,
    schema_version INTEGER NOT NULL DEFAULT 1,
    metadata JSONB NOT NULL DEFAULT '{}'
);
```

### Columns

| Column | Type | Description |
|--------|------|-------------|
| `id` | BIGSERIAL | Primary key (auto-incrementing integer) |
| `agent_id` | UUID | Agent reference, foreign key to agents(id) |
| `level` | reflection_level | Reflection level: 'micro', 'daily', or 'deep' |
| `created_at` | TIMESTAMPTZ | When this reflection was created |
| `session_id` | UUID | Session reference for micro reflections, NULL for daily/deep |
| `period_start` | TIMESTAMPTZ | Time window start for daily/deep reflections, NULL for micro |
| `period_end` | TIMESTAMPTZ | Time window end for daily/deep reflections, NULL for micro |
| `content` | TEXT | Main reflection content generated by LLM |
| `summary` | TEXT | Optional short title/summary |
| `experience_refs` | UUID[] | Array of experience IDs analyzed in this reflection |
| `reframing_note_ids` | UUID[] | Array of reframing note IDs produced by this reflection |
| `model_provider` | TEXT | LLM provider (e.g., 'ollama', 'anthropic') |
| `model_name` | TEXT | Model name (e.g., 'qwen3:14b') |
| `schema_version` | INTEGER | Schema version for migrations (currently 1) |
| `metadata` | JSONB | Additional metadata as JSONB object |

### Enum: `reflection_level`

```sql
CREATE TYPE reflection_level AS ENUM ('micro', 'daily', 'deep');
```

## Reflection Levels

### Micro Reflections

Micro reflections occur after each session to provide continuity and update the recent narrative layer.

**Characteristics:**
- `level = 'micro'`
- `session_id` is populated (references the session being reflected on)
- `period_start` and `period_end` are NULL
- Typically analyze experiences from a single session
- Focus on immediate learning and narrative updates

**Use case:** After-session summary, recent narrative layer update, immediate reframing

**Example query:**
```sql
SELECT * FROM public.reflections
WHERE level = 'micro'
  AND session_id = '123e4567-e89b-12d3-a456-426614174000'
ORDER BY created_at DESC;
```

### Daily Reflections

Daily reflections look for patterns across multiple sessions and detect recurring themes.

**Characteristics:**
- `level = 'daily'`
- `session_id` is NULL
- `period_start` and `period_end` define the time window (e.g., last 24 hours)
- Analyze experiences across multiple sessions
- Focus on pattern detection and habit formation

**Use case:** Pattern detection, habit identification, value reinforcement

**Example query:**
```sql
SELECT * FROM public.reflections
WHERE level = 'daily'
  AND agent_id = '123e4567-e89b-12d3-a456-426614174000'
  AND period_start >= NOW() - INTERVAL '7 days'
ORDER BY created_at DESC;
```

### Deep Reflections

Deep reflections occur weekly (or on schedule) for comprehensive narrative integration and health assessment.

**Characteristics:**
- `level = 'deep'`
- `session_id` is NULL
- `period_start` and `period_end` define a broader time window (e.g., last 7 days)
- Comprehensive analysis of identity, narrative, and psychological health
- May include Jahoda health criterion assessments

**Use case:** Narrative integration, health assessment, identity evolution tracking

**Example query:**
```sql
SELECT * FROM public.reflections
WHERE level = 'deep'
  AND agent_id = '123e4567-e89b-12d3-a456-426614174000'
ORDER BY created_at DESC
LIMIT 10;
```

## Indexes

The table includes the following indexes for efficient querying:

```sql
-- Timeline queries: get recent reflections for an agent
CREATE INDEX idx_reflections_agent_created
    ON public.reflections(agent_id, created_at DESC);

-- Level-filtered queries: get reflections by level
CREATE INDEX idx_reflections_level_created
    ON public.reflections(level, created_at);

-- Experience back-references: find all reflections that analyzed an experience
CREATE INDEX idx_reflections_experience_refs
    ON public.reflections USING GIN(experience_refs);
```

## Row Level Security (RLS)

The table uses Row Level Security to ensure agents can only access their own reflections:

```sql
ALTER TABLE public.reflections ENABLE ROW LEVEL SECURITY;

CREATE POLICY reflections_isolation ON public.reflections
    USING (agent_id = NULLIF(current_setting('atman.current_agent', TRUE), '')::UUID);
```

To set the agent context:
```sql
SET LOCAL atman.current_agent = '123e4567-e89b-12d3-a456-426614174000';
```

# Reflections Schema — Architecture Document

**Component:** Reflection Engine — PostgreSQL Storage Layer  
**Epic:** E27 — Core Reflections Schema  
**Status:** Implemented (v1)  
**Schema Version:** 1  

## Overview

The `public.reflections` table provides unified persistent storage for all three levels of reflection in Atman:

- **Micro**: After-session checkpoint (per session)
- **Daily**: End-of-day pattern detection (per calendar day)
- **Deep**: Scheduled deep reflection with health assessment (weekly/on-demand)

This document describes the schema, query patterns, RLS policies, and future extensions (notably: embedding-based search when E25 EmbeddingPort lands).

## Motivation

### Problem

Prior to E27, reflections were stored only in-memory via `ReflectionEventStore` (useful for testing, but not durable). The in-memory `ReflectionEvent` domain model captured internal workflow state (experiences analyzed, patterns detected, reframing notes added), but:

1. No persistent trail of reflection content generated by `OllamaReflectionModel`.
2. No way to query "what reflections were created for agent X in the past week?"
3. Blocked E0.3 eval (`reflection_quality` needs `FK → public.reflections(id)`).
4. Blocked Micro/Daily/DeepReflectionService persistence beyond in-memory prototypes.

### Solution

**E27** introduces `public.reflections` table:

- Unified schema for all three levels (discriminated by `level` enum).
- Session context (`session_id`) for micro; time period (`period_start`, `period_end`) for daily/deep.
- Free-form `content` field: stores LLM-generated reflection text.
- Reference arrays: `experience_refs` (experiences analyzed), `reframing_note_ids` (notes created).
- Model traceability: `model_provider`, `model_name` (e.g., `ollama` / `qwen3:14b`).
- RLS policy: `agent_id = current_setting('atman.current_agent')` (matches `facts`, `experiences`).
- Indexes: `(agent_id, created_at DESC)` for timeline, `GIN(experience_refs)` for cross-reference queries.

## Table Schema

```sql
CREATE TYPE reflection_level AS ENUM ('micro', 'daily', 'deep');

CREATE TABLE reflections (
    id                   BIGSERIAL PRIMARY KEY,
    agent_id             UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    level                reflection_level NOT NULL,
    created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    -- Session context (micro only)
    session_id           UUID REFERENCES sessions(id) ON DELETE SET NULL,
    
    -- Time period context (daily/deep only)
    period_start         TIMESTAMPTZ,
    period_end           TIMESTAMPTZ,
    
    -- Content
    content              TEXT NOT NULL,
    summary              TEXT,
    
    -- References
    experience_refs      UUID[] NOT NULL DEFAULT '{}',
    reframing_note_ids   UUID[] NOT NULL DEFAULT '{}',
    
    -- Model metadata
    model_provider       TEXT,
    model_name           TEXT,
    
    -- Schema versioning
    schema_version       INTEGER NOT NULL DEFAULT 1,
    metadata             JSONB NOT NULL DEFAULT '{}'
);
```

### Field Descriptions

| Field               | Type              | Description |
|---------------------|-------------------|-------------|
| `id`                | `BIGSERIAL`       | Primary key (auto-increment) |
| `agent_id`          | `UUID`            | FK → `agents.id` (CASCADE delete) |
| `level`             | `reflection_level`| `micro`/`daily`/`deep` |
| `created_at`        | `TIMESTAMPTZ`     | When reflection was generated (default: NOW()) |
| `session_id`        | `UUID`            | FK → `sessions.id`; populated for `level='micro'`, NULL otherwise |
| `period_start`      | `TIMESTAMPTZ`     | Start of analyzed period (daily/deep); NULL for micro |
| `period_end`        | `TIMESTAMPTZ`     | End of analyzed period (daily/deep); NULL for micro |
| `content`           | `TEXT`            | Free-form reflection text generated by LLM (NOT NULL) |
| `summary`           | `TEXT`            | Optional short title/summary |
| `experience_refs`   | `UUID[]`          | Array of `experiences.id` analyzed in this reflection |
| `reframing_note_ids`| `UUID[]`          | Array of `reframing_notes.id` created by this reflection |
| `model_provider`    | `TEXT`            | LLM provider: `ollama`, `anthropic`, etc. |
| `model_name`        | `TEXT`            | Specific model: e.g., `qwen3:14b`, `claude-3.5-sonnet` |
| `schema_version`    | `INTEGER`         | Schema version for migrations (current: 1) |
| `metadata`          | `JSONB`           | Additional structured data (e.g., health assessment references) |

### Indexes

```sql
CREATE INDEX idx_reflections_agent_time ON reflections(agent_id, created_at DESC);
CREATE INDEX idx_reflections_level_time ON reflections(level, created_at DESC);
CREATE INDEX idx_reflections_experience_refs ON reflections USING GIN(experience_refs);
CREATE INDEX idx_reflections_session ON reflections(session_id) WHERE session_id IS NOT NULL;
```

**Purpose:**

- `idx_reflections_agent_time`: Timeline queries ("most recent reflections for agent X")
- `idx_reflections_level_time`: Level-filtered queries ("all daily reflections, newest first")
- `idx_reflections_experience_refs`: Cross-reference ("which reflections analyzed experience Y?")
- `idx_reflections_session`: Session lookup (micro reflections)

### RLS Policy

```sql
ALTER TABLE reflections ENABLE ROW LEVEL SECURITY;

CREATE POLICY reflections_isolation ON reflections
    USING (agent_id = NULLIF(current_setting('atman.current_agent', TRUE), '')::UUID);
```

**Behavior:**

- Session must `SET atman.current_agent = '<agent-uuid>'` before SELECT/INSERT/UPDATE/DELETE.
- Only reflections for the current agent are visible.
- Bypassed for superuser / roles with `BYPASS RLS` (e.g., `atman_eval_reader` for E0.3 eval scripts).

## Query Patterns

### Get all reflections for an agent

```sql
SELECT id, level, created_at, summary, content
FROM public.reflections
WHERE agent_id = $1
ORDER BY created_at DESC;
```

### Get reflections for a specific session

```sql
SELECT id, level, created_at, content
FROM public.reflections
WHERE session_id = $1
ORDER BY created_at DESC;
```

### Find reflections that analyzed a specific experience

```sql
SELECT id, level, created_at, summary, content
FROM public.reflections
WHERE $1 = ANY(experience_refs)
ORDER BY created_at DESC;
```

### Get recent reflections by level

```sql
SELECT id, created_at, summary, content
FROM public.reflections
WHERE agent_id = $1 AND level = $2
ORDER BY created_at DESC
LIMIT $3;
```

### Time-series analysis of reflection frequency

```sql
SELECT
    DATE_TRUNC('day', created_at) as day,
    level,
    COUNT(*) as reflection_count
FROM public.reflections
WHERE agent_id = $1
  AND created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE_TRUNC('day', created_at), level
ORDER BY day DESC, level;
```

## Three Reflection Levels

### Micro (After-Session)

- **Trigger:** End of each session
- **Scope:** Single session
- **Purpose:** Update recent narrative layer; checkpoint session state
- **Fields populated:**
  - `level = 'micro'`
  - `session_id` (NOT NULL)
  - `period_start`, `period_end` (NULL)
  - `experience_refs` (experiences from this session)

### Daily (End-of-Day)

- **Trigger:** End of calendar day (UTC)
- **Scope:** All sessions within the day
- **Purpose:** Pattern detection across sessions; update user/relationship context
- **Fields populated:**
  - `level = 'daily'`
  - `session_id` (NULL)
  - `period_start` (start of day UTC: 00:00:00)
  - `period_end` (end of day UTC: 23:59:59)
  - `experience_refs` (experiences from all sessions that day)
  - `reframing_note_ids` (if reframing notes were added)

### Deep (Scheduled / On-Demand)

- **Trigger:** Weekly schedule or manual request
- **Scope:** Past week (or custom period)
- **Purpose:** Health assessment (Jahoda criteria), narrative revision, identity snapshot proposals
- **Fields populated:**
  - `level = 'deep'`
  - `session_id` (NULL)
  - `period_start`, `period_end` (analyzed time window)
  - `experience_refs` (experiences from period)
  - `reframing_note_ids` (if deep reflection produced notes)
  - `metadata` (may contain `health_assessment_id`, `identity_snapshot_id`)

### Embedding Column (E25 EmbeddingPort)

Once E25 lands, add:

**Purpose:** Semantic search over reflection content ("find reflections similar to X").

**Migration path:** Documented in E25 epic; embedding generation will be async background job (not blocking reflection creation).

### Reflection Threads

Future consideration: link related reflections across time via `thread_id` or `parent_reflection_id` (e.g., daily reflection references yesterday's reflection). Not in v1 schema.

### Identity Proposal References

Deep reflections may propose identity changes. Schema v1 stores this in `metadata` JSONB. Future: separate `identity_proposals` table with FK to `reflections.id`.

## Troubleshooting

### psycopg not installed

If you see `ImportError: psycopg is required for ReflectionStore`, install it:

```bash
pip install psycopg[binary]
```

Or use the optional dependency:

```bash
pip install "atman[postgres]"
```

### RLS policy blocking access

If queries return no results, check that the agent context is set:

```python
import os
os.environ["ATMAN_CURRENT_AGENT"] = str(agent_uuid)
```

Or in SQL:

```sql
SET LOCAL atman.current_agent = 'your-agent-uuid';
```

### Connection errors

Ensure the database URL is configured:

```bash
export ATMAN_DB_URL="postgresql://atman@localhost:5432/atman"
