# Reflections Schema — Architecture Document

**Component:** Reflection Engine — PostgreSQL Storage Layer  
**Epic:** E27 — Core Reflections Schema  
**Status:** Implemented (v1)  
**Schema Version:** 1  

## Overview

The `public.reflections` table provides unified persistent storage for all three levels of reflection in Atman:

- **Micro**: After-session checkpoint (per session)
- **Daily**: End-of-day pattern detection (per calendar day)
- **Deep**: Scheduled deep reflection with health assessment (weekly/on-demand)

This document describes the schema, query patterns, RLS policies, and future extensions (notably: embedding-based search when E25 EmbeddingPort lands).

## Motivation

### Problem

Prior to E27, reflections were stored only in-memory via `ReflectionEventStore` (useful for testing, but not durable). The in-memory `ReflectionEvent` domain model captured internal workflow state (experiences analyzed, patterns detected, reframing notes added), but:

1. No persistent trail of reflection content generated by `OllamaReflectionModel`.
2. No way to query "what reflections were created for agent X in the past week?"
3. Blocked E0.3 eval (`reflection_quality` needs `FK → public.reflections(id)`).
4. Blocked Micro/Daily/DeepReflectionService persistence beyond in-memory prototypes.

### Solution

**E27** introduces `public.reflections` table:

- Unified schema for all three levels (discriminated by `level` enum).
- Session context (`session_id`) for micro; time period (`period_start`, `period_end`) for daily/deep.
- Free-form `content` field: stores LLM-generated reflection text.
- Reference arrays: `experience_refs` (experiences analyzed), `reframing_note_ids` (notes created).
- Model traceability: `model_provider`, `model_name` (e.g., `ollama` / `qwen3:14b`).
- RLS policy: `agent_id = current_setting('atman.current_agent')` (matches `facts`, `experiences`).
- Indexes: `(agent_id, created_at DESC)` for timeline, `GIN(experience_refs)` for cross-reference queries.

## Table Schema

```sql
CREATE TYPE reflection_level AS ENUM ('micro', 'daily', 'deep');

CREATE TABLE reflections (
    id                   BIGSERIAL PRIMARY KEY,
    agent_id             UUID NOT NULL REFERENCES agents(id) ON DELETE CASCADE,
    level                reflection_level NOT NULL,
    created_at           TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    
    -- Session context (micro only)
    session_id           UUID REFERENCES sessions(id) ON DELETE SET NULL,
    
    -- Time period context (daily/deep only)
    period_start         TIMESTAMPTZ,
    period_end           TIMESTAMPTZ,
    
    -- Content
    content              TEXT NOT NULL,
    summary              TEXT,
    
    -- References
    experience_refs      UUID[] NOT NULL DEFAULT '{}',
    reframing_note_ids   UUID[] NOT NULL DEFAULT '{}',
    
    -- Model metadata
    model_provider       TEXT,
    model_name           TEXT,
    
    -- Schema versioning
    schema_version       INTEGER NOT NULL DEFAULT 1,
    metadata             JSONB NOT NULL DEFAULT '{}'
);
```

### Field Descriptions

| Field               | Type              | Description |
|---------------------|-------------------|-------------|
| `id`                | `BIGSERIAL`       | Primary key (auto-increment) |
| `agent_id`          | `UUID`            | FK → `agents.id` (CASCADE delete) |
| `level`             | `reflection_level`| `micro`/`daily`/`deep` |
| `created_at`        | `TIMESTAMPTZ`     | When reflection was generated (default: NOW()) |
| `session_id`        | `UUID`            | FK → `sessions.id`; populated for `level='micro'`, NULL otherwise |
| `period_start`      | `TIMESTAMPTZ`     | Start of analyzed period (daily/deep); NULL for micro |
| `period_end`        | `TIMESTAMPTZ`     | End of analyzed period (daily/deep); NULL for micro |
| `content`           | `TEXT`            | Free-form reflection text generated by LLM (NOT NULL) |
| `summary`           | `TEXT`            | Optional short title/summary |
| `experience_refs`   | `UUID[]`          | Array of `experiences.id` analyzed in this reflection |
| `reframing_note_ids`| `UUID[]`          | Array of `reframing_notes.id` created by this reflection |
| `model_provider`    | `TEXT`            | LLM provider: `ollama`, `anthropic`, etc. |
| `model_name`        | `TEXT`            | Specific model: e.g., `qwen3:14b`, `claude-3.5-sonnet` |
| `schema_version`    | `INTEGER`         | Schema version for migrations (current: 1) |
| `metadata`          | `JSONB`           | Additional structured data (e.g., health assessment references) |

### Indexes

```sql
CREATE INDEX idx_reflections_agent_time ON reflections(agent_id, created_at DESC);
CREATE INDEX idx_reflections_level_time ON reflections(level, created_at DESC);
CREATE INDEX idx_reflections_experience_refs ON reflections USING GIN(experience_refs);
CREATE INDEX idx_reflections_session ON reflections(session_id) WHERE session_id IS NOT NULL;
```

**Purpose:**

- `idx_reflections_agent_time`: Timeline queries ("most recent reflections for agent X")
- `idx_reflections_level_time`: Level-filtered queries ("all daily reflections, newest first")
- `idx_reflections_experience_refs`: Cross-reference ("which reflections analyzed experience Y?")
- `idx_reflections_session`: Session lookup (micro reflections)

### RLS Policy

```sql
ALTER TABLE reflections ENABLE ROW LEVEL SECURITY;

CREATE POLICY reflections_isolation ON reflections
    USING (agent_id = NULLIF(current_setting('atman.current_agent', TRUE), '')::UUID);
```

**Behavior:**

- Session must `SET atman.current_agent = '<agent-uuid>'` before SELECT/INSERT/UPDATE/DELETE.
- Only reflections for the current agent are visible.
- Bypassed for superuser / roles with `BYPASS RLS` (e.g., `atman_eval_reader` for E0.3 eval scripts).

## Usage Patterns

### Store a Micro Reflection

```python
from atman.adapters.storage.in_memory_postgres_reflection_store import InMemoryReflectionStore
from atman.adapters.storage.reflection_persistence_helper import persist_micro_reflection

store = InMemoryReflectionStore()  # or PostgreSQL adapter when available

record = persist_micro_reflection(
    store,
    agent_id=agent_id,
    session_id=session_id,
    content="Session completed successfully. User was collaborative.",
    summary="Positive session",
    experience_refs=[exp_id1, exp_id2],
    model_provider="ollama",
    model_name="qwen3:14b",
)

print(f"Stored reflection id={record.id}")
```

### Query Recent Reflections

```python
recent = store.list_recent(agent_id, limit=10)
for r in recent:
    print(f"{r.level} — {r.created_at} — {r.summary}")
```

### Query Reflections by Level

```python
from datetime import datetime, timedelta
from atman.core.models.reflection import ReflectionLevel

cutoff = datetime.now(UTC) - timedelta(days=7)
daily_reflections = store.list_by_level(agent_id, ReflectionLevel.DAILY, since=cutoff)
```

### Cross-Reference: Which Reflections Analyzed an Experience?

```python
reflections = store.list_by_experience(experience_id)
for r in reflections:
    print(f"Reflection {r.id} ({r.level}) analyzed this experience")
```

## Three Reflection Levels

### Micro (After-Session)

- **Trigger:** End of each session
- **Scope:** Single session
- **Purpose:** Update recent narrative layer; checkpoint session state
- **Fields populated:**
  - `level = 'micro'`
  - `session_id` (NOT NULL)
  - `period_start`, `period_end` (NULL)
  - `experience_refs` (experiences from this session)

### Daily (End-of-Day)

- **Trigger:** End of calendar day (UTC)
- **Scope:** All sessions within the day
- **Purpose:** Pattern detection across sessions; update user/relationship context
- **Fields populated:**
  - `level = 'daily'`
  - `session_id` (NULL)
  - `period_start` (start of day UTC: 00:00:00)
  - `period_end` (end of day UTC: 23:59:59)
  - `experience_refs` (experiences from all sessions that day)
  - `reframing_note_ids` (if reframing notes were added)

### Deep (Scheduled / On-Demand)

- **Trigger:** Weekly schedule or manual request
- **Scope:** Past week (or custom period)
- **Purpose:** Health assessment (Jahoda criteria), narrative revision, identity snapshot proposals
- **Fields populated:**
  - `level = 'deep'`
  - `session_id` (NULL)
  - `period_start`, `period_end` (analyzed time window)
  - `experience_refs` (experiences from period)
  - `reframing_note_ids` (if deep reflection produced notes)
  - `metadata` (may contain `health_assessment_id`, `identity_snapshot_id`)

## Integration with Other Components

### OllamaReflectionModel

`OllamaReflectionModel` generates structured reflection content (narrative updates, reframing notes, patterns). Helper functions in `reflection_persistence_helper.py` convert LLM outputs to `ReflectionRecord` and persist via `ReflectionStore`:

```python
from atman.adapters.reflection.ollama_reflection_model import OllamaReflectionModel
from atman.adapters.storage.reflection_persistence_helper import persist_micro_reflection

model = OllamaReflectionModel()
# ... generate reflection content via model.propose_narrative_update(...)

persist_micro_reflection(
    store,
    agent_id=agent_id,
    session_id=session_id,
    content=generated_narrative_update,
    model_provider="ollama",
    model_name=model.model,
)
```

### E0.3 Eval (reflection_quality)

E0.3 quality assessment schema includes:

```sql
CREATE TABLE eval.reflection_quality (
    reflection_id BIGINT NOT NULL REFERENCES public.reflections(id),
    ...
);
```

This FK unblocks automated quality checks on reflections.

### Micro/Daily/DeepReflectionService

Future work: services will call `ReflectionStore.add()` after generating reflections. Currently services use in-memory `ReflectionEventStore`. E27 provides the schema; service integration is follow-up work (tracked in MODEL-02, MODEL-03 backlog).

## Future Extensions

### Embedding Column (E25 EmbeddingPort)

Once E25 lands, add:

```sql
ALTER TABLE reflections ADD COLUMN embedding VECTOR(768);
ALTER TABLE reflections ADD COLUMN embed_model TEXT;

CREATE INDEX idx_reflections_embedding
    ON reflections USING hnsw(embedding vector_cosine_ops);
```

**Purpose:** Semantic search over reflection content ("find reflections similar to X").

**Migration path:** Documented in E25 epic; embedding generation will be async background job (not blocking reflection creation).

### Reflection Threads

Future consideration: link related reflections across time via `thread_id` or `parent_reflection_id` (e.g., daily reflection references yesterday's reflection). Not in v1 schema.

### Identity Proposal References

Deep reflections may propose identity changes. Schema v1 stores this in `metadata` JSONB. Future: separate `identity_proposals` table with FK to `reflections.id`.

## Known Limitations (v1)

1. **No real PostgreSQL adapter yet:** E27 provides schema + in-memory implementation. Real `psycopg`-based adapter is follow-up work (requires adding `psycopg` dependency).
2. **No semantic search:** Text search only (`pg_trgm` on `content`). Embedding-based search requires E25.
3. **Service integration incomplete:** `OllamaReflectionModel` generates content; helper functions exist to persist; but `MicroReflectionService`/`DailyReflectionService`/`DeepReflectionService` still use in-memory `ReflectionEventStore`. Wiring services to new `ReflectionStore` is tracked in MODEL-02, MODEL-03 backlog.
