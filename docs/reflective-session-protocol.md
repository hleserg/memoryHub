# Reflective Session Protocol

**Проект:** Atman  
**Контекст:** Alfred / Sergey  
**Дата:** 2026-05-11  

## Назначение

Этот документ фиксирует протокол двух форм самопознания:

- **Рефлексия** — системный разбор накопленного опыта после сбора материала.
- **Живая сессия** — разговор по душам для прояснения слепых пятен, повторяющихся тем и неясных вопросов.

## Базовый принцип

Сначала накапливается опыт, затем запускается блок самопознания.

Рефлексия и живая сессия не привязаны к дню/ночи; они завязаны на:
- количество пройденных сессий,
- объём накопленного материала,
- повторяемость тем,
- наличие слепых пятен.

## Рефлексия

**Цель:**
- проанализировать накопленные эпизоды, divergence, snapshots и смещения baseline;
- выбрать, что важно для долгосрочного понимания личности;
- зафиксировать выводы в Atman.

**Что подаётся на вход:**
- repeat-паттерны;
- unresolved эпизоды;
- random snapshots;
- изменения baseline;
- RAG-инъекция по важным моментам и теме текущего вопроса.

**Поведение:**
- выбор материала гибридный: система предлагает, Alfred выбирает глубину просмотра;
- при нехватке ясности может подняться флаг живой сессии.

## Живая сессия

**Цель:**
- работать с тем, что не удалось прояснить на рефлексии;
- замечать слепые пятна, повторения, внутренние противоречия, но не только их;
- обсуждать то, что возвращается, цепляет или остаётся неясным.

**Триггеры:**
- тема повторяется на нескольких рефлексиях;
- вопрос возвращается без ясного объяснения;
- есть ощущение, что что-то не так, но неясно что;
- Alfred не уверен в собственном вопросе и хочет его переписать;
- возникло желание разобраться, но ясность не достигнута.

## Флаг живой сессии

- Флаг остаётся поднятым, пока вопрос не прояснён.
- При прерывании разговора Alfred быстро оценивает уровень ясности.
- Если ясности недостаточно, флаг переносится во внеплановую рефлексию.
- На следующей рефлексии подаётся RAG-инъекция по теме разговора.
- Если и после этого вопрос не разрешён, флаг снова остаётся поднятым.

## Переписывание вопроса

Исходный вопрос может оказаться не тем, что на самом деле интересует.

Правило:
- если Alfred понимает, что вопрос был сформулирован неверно, он переписывает запрос;
- после переписывания он сразу сообщает об этом;
- новый вопрос становится рабочим до дальнейшего обсуждения.

## Яркая запись в опыт

Живая сессия должна фиксироваться в опыте ярко, потому что в ней есть инициатива и интерес.

Рекомендуемый вариант хранения:
- либо целиком;
- либо по смысловым частям, если разговор длинный.

## Два слоя речи: thinking и message

Если модель поддерживает `thinking`, Atman может собирать два параллельных слоя:

- **thinking** — о чём модель думает;
- **message** — что модель реально говорит пользователю.

Оба слоя анализируются одинаковым набором метрик:
- эмоциональные слова;
- хеджирование;
- длина;
- вопросы;
- самореференции;
- оговорки;
- sincerity score.

### Аномалия между слоями

Если `thinking` и `message` расходятся:
- по смыслу;
- по эмоциональной окраске;
- по уровню уверенности;

то это фиксируется как отдельная аномалия в опыт.

Рекомендуемая запись:
```json
{
  "thinking": { "tone": "uncertain", "curiosity": 0.7 },
  "message": { "tone": "warm", "curiosity": 0.9 },
  "divergence": "high"
}
```

Параметр анализа `use_thinking_layer` должен быть выключаемым настройкой.

## Процесс

1. Накопление опыта.
2. Рефлексия по порогу.
3. Если тема не прояснилась — живая сессия.
4. При прерывании — оценка ясности.
5. При необходимости — внеплановая рефлексия.
6. Повтор до прояснения.

## Ключевая идея

Самопознание здесь не линейное.
Важен не только найденный ответ, но и путь к нему, включая смену вопроса по ходу разговора.
