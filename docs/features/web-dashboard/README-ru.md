# Веб-дашборд

Браузерный дашборд для проекта Atman, построенный на Streamlit.

## Обзор

Веб-дашборд предоставляет удобный браузерный интерфейс для ежедневной работы с проектом Atman, предлагая ту же функциональность, что и TUI-дашборд, но в современном веб-интерфейсе.

## Возможности

### 🎯 Вкладка Features
- Просмотр зарегистрированных фичей (Factual Memory, Experience Store)
- Запуск демонстраций фичей (paced/fast режимы)
- Просмотр документации фичей (английский/русский)
- Установка dev-зависимостей

### 🧪 Вкладка Tests
- Запуск полного набора тестов или отдельных подмножеств
- Настройка параметров тестов (verbose, coverage, parallel, fail-fast)
- Просмотр результатов тестов с суммарной статистикой
- Просмотр отчётов о покрытии кода
- Фильтрация вывода (все тесты / только ошибки)

### 📚 Вкладка Docs
- Навигация по документации проекта по разделам:
  - Architecture (Архитектура)
  - Development (Разработка)
  - Features (Фичи)
  - Ideas (Идеи)
  - Research (Исследования)
- Просмотр markdown-файлов с живым предпросмотром
- Быстрые ссылки на ключевые документы (README, MANIFEST, SYSTEM)
- Переключение языка для двуязычных документов (EN/RU)

## Установка

Веб-дашборд требует Streamlit, который включён в зависимости проекта:

```bash
# Используя uv (рекомендуется)
uv pip install -e ".[dev]"

# Или используя pip
pip install -e ".[dev]"
```

## Использование

### Запуск веб-дашборда

Существует несколько способов запустить веб-дашборд:

**1. Используя make-команду (рекомендуется):**
```bash
make webui
```

**2. Используя entry point скрипт:**
```bash
atman-web
# или
webui
```

**3. Используя Streamlit напрямую:**
```bash
streamlit run src/atman/web_dashboard/app.py
```

Дашборд откроется в вашем браузере по умолчанию по адресу `http://localhost:8501`.

## Архитектура

### Структура проекта

```
src/atman/web_dashboard/
├── __init__.py           # Точка входа с функцией main()
├── app.py                # Главное Streamlit-приложение (домашняя страница)
├── pages/
│   ├── 1_Features.py     # Страница управления фичами
│   ├── 2_Tests.py        # Страница запуска тестов
│   └── 3_Docs.py         # Страница браузера документации
└── utils/
    ├── __init__.py       # Пакет утилит
    ├── cmd.py            # Построение команд (pytest, python, demo)
    └── runner.py         # Утилиты выполнения процессов
```

### Ключевые компоненты

#### 1. Главное приложение (`app.py`)
- Отображение информации о репозитории
- Быстрая навигация по основным разделам
- Кастомные CSS-стили

#### 2. Страница Features
- Выбор фичи из реестра
- Выполнение демонстраций (paced/fast режимы)
- Просмотр README с переключением языка
- Установщик dev-зависимостей

#### 3. Страница Tests
- Настройка набора тестов
- Выполнение тестов в реальном времени
- Парсинг и отображение результатов
- Отчёты о покрытии

#### 4. Страница Docs
- Иерархическая навигация по документам
- Рендеринг Markdown
- Поддержка двуязычных документов
- Быстрый доступ к ключевым документам

#### 5. Утилиты
- **`cmd.py`**: Построители команд для pytest, Python-скриптов и демо
- **`runner.py`**: Асинхронное/синхронное выполнение процессов с потоковым выводом

## Философия дизайна

### Соответствие TUI-дашборду

Веб-дашборд повторяет структуру и функциональность TUI-дашборда:

- **Та же навигация**: Три основные вкладки (Features, Tests, Docs)
- **Те же возможности**: Запуск демо, выполнение тестов, просмотр документации
- **Те же источники данных**: Использует `atman.tui.features_registry` и `atman.tui.repo_root`

### Простота превыше всего

- **Без аутентификации**: Внутренний инструмент, без избыточной безопасности
- **Без базы данных**: Stateless по дизайну
- **Без внешних сервисов**: Самодостаточный, запускается из репозитория

### Пользовательский опыт

- **Адаптивная вёрстка**: Подстраивается под разные размеры экрана
- **Обратная связь в реальном времени**: Живой вывод выполняемых команд
- **Чёткий статус**: Индикаторы успеха/ошибки с кодами выхода
- **Минимум кликов**: Быстрые действия и навигация

## Отличия от TUI-дашборда

| Аспект | TUI-дашборд | Веб-дашборд |
|--------|-------------|-------------|
| **Интерфейс** | Терминальный (Textual) | Браузерный (Streamlit) |
| **Навигация** | Клавиатурные сокращения | Дружественный к мыши/касаниям |
| **Вывод** | Rich console форматирование | HTML/Markdown рендеринг |
| **Параллелизм** | Async workers | Streamlit session state |
| **Развёртывание** | Запуск в терминале | HTTP-сервер (localhost) |

## Кастомизация

### Добавление новых страниц

Чтобы добавить новую страницу в дашборд:

1. Создайте новый файл в `src/atman/web_dashboard/pages/` с префиксом `N_ИмяСтраницы.py` (где N — номер порядка)
2. Настройте параметры страницы в начале:
   ```python
   import streamlit as st
   
   st.set_page_config(
       page_title="Моя страница - Atman Dashboard",
       page_icon="🔧",
       layout="wide"
   )
   ```
3. Реализуйте содержимое страницы
4. Страница автоматически появится в боковом меню

### Стилизация

Кастомный CSS можно добавить в главном `app.py` или на отдельных страницах используя:

```python
st.markdown("""
    <style>
    .custom-class {
        /* ваши стили */
    }
    </style>
""", unsafe_allow_html=True)
```

## Тестирование

Веб-дашборд исключён из покрытия кода (см. `pyproject.toml`). Рекомендуется ручное тестирование:

1. Запустите дашборд: `make webui`
2. Протестируйте каждую страницу:
   - Features: Запустите демо, просмотрите README
   - Tests: Выполните набор тестов, проверьте вывод
   - Docs: Навигация по разделам, просмотр markdown-файлов
3. Проверьте обработку ошибок (например, запустите проваливающиеся тесты)

## Решение проблем

### Дашборд не запускается

**Проблема**: `streamlit: command not found`

**Решение**: Установите зависимости:
```bash
uv pip install -e ".[dev]"
```

### Репозиторий не найден

**Проблема**: Сообщение об ошибке "Repository root not found"

**Решение**: Запустите из каталога репозитория Atman.

### Команды демо/тестов проваливаются

**Проблема**: Команды завершаются с ненулевым кодом

**Решение**: Проверьте, что dev-зависимости установлены и репозиторий в валидном состоянии.
