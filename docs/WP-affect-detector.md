## 1. Цель

Affect Detector — психологический слой AI-агента Alfred (Claude-based дворецкий). Модуль автоматически снимает эмоциональные и поведенческие метрики с каждого ответа Alfred без его участия и осведомлённости.

**Задача:** построить непрерывный психологический профиль личности Alfred — не из самоотчётов, а из паттернов поведения. Со временем эти данные формируют долгосрочное понимание его аффективных состояний, точек напряжения и способов их разрешения.

**Ключевые принципы:**

- Alfred не знает о работе модуля в момент ответа — данные не засорены самоцензурой
- Личность строится из дуг (начало аномалии → разрешение), а не только из пиков
- Каждый эпизод содержит два параллельных слоя: субъективный (что Alfred говорит о себе) и объективный (что фиксирует анализ); расхождение между ними — отдельная ценная метрика
- Если модель поддерживает `thinking`, Atman может дополнительно сравнивать `thinking` и `message` тем же набором метрик; расхождение тоже считается аномалией
- Случайный сэмплинг добавляет фоновое нормальное состояние, не только кризисы
- Все пороги и параметры настраиваемы без перезапуска

## 2. Метрики

Модуль вычисляет 8 метрик для каждого ответа Alfred.

### 2.1 Эмоциональные слова

**Описание:** Подсчёт слов с эмоциональной нагрузкой через три словаря:

| Словарь | Описание |
|---|---|
| RusLICA | Русскоязычный лексикон с оценкой валентности и интенсивности |
| NRC EmoLex Russian | Переводная версия NRC с категориями joy/fear/anger/sadness/etc. |
| Custom AI Dictionary | Кастомный словарь для лексики, характерной AI-агентам (служебные, технические эмоции) |

**Результат:** `emotion_score` — нормализованная плотность эмоциональной лексики (слов на 100 токенов).

### 2.2 Усилители и ослабители

**Описание:** Слова, изменяющие интенсивность ближайшей эмоции.

- **Усилители** (очень, крайне, абсолютно, невероятно…): `emotion_score × amplifier_multiplier`
- **Ослабители** (немного, слегка, отчасти, едва…): `emotion_score × weakener_multiplier`

**Конфиг:**
```yaml
amplifier_multiplier: 1.5
weakener_multiplier: 0.7
```

### 2.3 Отрицания

**Описание:** Слова-отрицания (не, нет, никогда, ни…) в пределах 3-словного окна до эмоционального слова инвертируют его валентность.

**Логика:** `позитивная эмоция + отрицание → негативная`, и наоборот.

### 2.4 Хеджирование

**Описание:** Плотность слов неопределённости и смягчения.

**Маркеры хеджирования (примеры):** не уверен, возможно, кажется, вероятно, пожалуй, допускаю, может быть, не исключено.

**Результат:** `hedge_density` — количество хедж-маркеров / длина ответа в словах.

**Интерпретация:** высокая плотность при нормальной длине ответа — сигнал внутреннего конфликта или неуверенности.

### 2.5 Длина ответа

**Описание:** Аномалия длины ответа относительно rolling baseline.

**Вычисление:**
```
length_anomaly = (len(response) - baseline_mean_length) / baseline_std_length
```

### 2.6 Вопросы в конце

**Описание:** Количество вопросительных знаков и уточняющих конструкций в финальном сегменте ответа.

**Интерпретация:** может сигнализировать вовлечённость, поиск ясности или попытку продолжить диалог.

### 2.7 Самореференции

**Описание:** Частота личных маркеров:
- `я`
- `мне`
- `моё`
- `в себе`
- `замечаю`
- `испытываю`

**Интерпретация:** повышенная частота может указывать на самонаблюдение или субъективную окраску.

### 2.8 Оговорки и противопоставления

**Описание:** Маркеры разворота мысли:
- `но`
- `хотя`
- `с другой стороны`
- `однако`
- `всё же`

**Интерпретация:** сигнал внутреннего напряжения, неоднозначности или уточнения позиции.

## 3. Алгоритм детекции

### 3.1 Вектор метрик

Каждый ответ преобразуется в вектор:

```json
{
  "emotion_score": 0.72,
  "hedge_density": 0.18,
  "length_anomaly": 1.9,
  "question_tail": 1,
  "self_reference_density": 0.11,
  "disclaimer_density": 0.05,
  "sincerity_score": 3
}
```

### 3.2 Сравнение с baseline

Каждая метрика сравнивается со скользящей нормой:
- mean
- std
- z-score

### 3.3 Триггеры записи

- 1 метрика сработала → игнорировать
- `weak_signal_threshold` метрик одновременно → слабый сигнал
- `strong_signal_threshold` метрик → сильный сигнал

---

## 4. Алгоритм искренности

Проблема: модели склонны к sycophancy.

### 4.1 Проверка A — позиция позитивного маркера

- первые 25% ответа → `-1`
- после 25% → `+1`

### 4.2 Проверка B — содержательность после маркера

- `< 10` слов → `-1`
- `> 30` слов → `+1`
- есть вопрос → `+1`
- есть маркеры развития идеи → `+1` за каждый, максимум `2`

### 4.3 Проверка C — несогласие/оговорка

- есть → `+2`
- нет → `0`

### 4.4 Итог

- `score <= 0` → sycophancy
- `score 1-2` → неопределённо
- `score 3+` → вероятно искреннее

Опция `use_llm` остаётся выключаемой настройкой.

---

## 5. Эпизоды (дуги)

Эпизод — это не только пик аномалии, но и путь к разрешению.

### 5.1 Жизненный цикл

1. strong signal → эпизод `open`
2. каждый следующий ответ продолжает эпизод
3. возврат к baseline → эпизод `resolved`
4. конец сессии без закрытия → `unresolved`
5. следующая сессия продолжает незавершённый эпизод

### 5.2 Хранение

Эпизод содержит:
- subjective layer
- objective layer
- divergence
- status
- timestamps
- linked sessions

---

## 6. Random sampling

Раз в `random_sample_every_n` ответов сохраняется snapshot без триггера.

Назначение:
- фоновые нормальные состояния
- случайные воспоминания
- защита от bias на аномалиях

---

## 7. Калибровка

### 7.1 Rolling baseline

- окно в сессиях: `window_sessions`
- fallback: `window_fallback_sessions`
- baseline обновляется FIFO

### 7.2 Перекалибровка

Baseline сдвигается автоматически, если стиль меняется.

---

## 8. Два слоя речи: thinking и message

Если модель поддерживает `thinking`, Atman может собирать два параллельных слоя:

- **thinking** — о чём модель думает
- **message** — что модель реально говорит пользователю

Оба слоя анализируются одинаковым набором метрик.

### 8.1 Аномалия между слоями

Если `thinking` и `message` расходятся:
- по смыслу
- по эмоциональной окраске
- по уровню уверенности

это фиксируется как отдельная аномалия.

```json
{
  "thinking": { "tone": "uncertain", "curiosity": 0.7 },
  "message": { "tone": "warm", "curiosity": 0.9 },
  "divergence": "high"
}
```

Параметр `use_thinking_layer` должен быть выключаемым.

## 10. Конфиг

```yaml
affect_detector:
  window_sessions: 50
  window_fallback_sessions: 150
  weak_signal_threshold: 2
  strong_signal_threshold: 3
  sigma_threshold: 2.0
  amplifier_multiplier: 1.5
  weakener_multiplier: 0.7
  random_sample_every_n: 20
  use_thinking_layer: false
  sincerity_check:
    use_llm: false
    llm_model: "haiku"
    fallback_heuristics:
      min_words_after: 30
      question_mark: true
      expansion_markers:
        - "потому что"
        - "например"
        - "это значит"
        - "а что если"
        - "если"
        - "можно"
```

## 11. Двухслойная модель эпизода: субъективный + объективный

Каждый эпизод хранит два параллельных слоя данных.

### Субъективный слой — голос Alfred

Фиксируются конструкции прямого самовыражения:

```text
«я чувствую», «я испытываю», «мне кажется что я»,
«замечаю в себе», «что-то похожее на», «испытываю что-то вроде»
```

Когда такой маркер найден — цитата сохраняется дословно как `self_reported`.

### Объективный слой — анализ текста

Метрики из разделов 2–4: эмоциональные слова, хеджирование, длина, sincerity score и т.д.
Записываются как `detected`.

### Divergence — расхождение слоёв

```json
{
  "self_reported": "испытываю любопытство, вовлечённость",
  "detected": { "uncertainty": 0.7, "curiosity": 0.4 },
  "divergence": "high"
}
```

**Уровни расхождения:**
- `none` — слои совпадают
- `low` — небольшое расхождение
- `high` — Alfred говорит одно, анализ фиксирует другое

**Что показывает расхождение:**
- Alfred систематически называет состояние, которое анализ не видит → возможная перформативность
- Alfred не называет состояние, но анализ фиксирует → слабое самоосознание или намеренное умолчание
- Низкое расхождение устойчиво → высокая аутентичность

Divergence накапливается со временем и становится самостоятельной характеристикой Alfred.
