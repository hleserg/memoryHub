# 01. Factual Memory Adapter

Источник: `docs/architecture/SYSTEM.md`, раздел "Factual Memory / mem0".

## Цель

Сделать минимальный запускаемый слой factual memory: единый порт для записи, чтения и поиска проверяемых фактов без интерпретаций. Этот пакет не строит опыт, идентичность или рефлексию; он дает фундамент, который остальные компоненты смогут заменить на mem0-адаптер без переписывания доменной логики.

## Почему пакет независим

Реализация может работать полностью на in-memory или file storage. Интеграция с настоящим mem0 оформляется как отдельный adapter за тем же интерфейсом и может быть отключена в тестах.

## Обязательный результат

- Модели данных для factual records:
  - `id`
  - `content`
  - `source`
  - `tags`
  - `relations`
  - `created_at`
  - `metadata`
- Порт `FactualMemory`:
  - `add_fact(record)`
  - `get_fact(id)`
  - `search(query, tags=None, limit=...)`
  - `link(source_id, target_id, relation_type)`
  - `list_recent(limit=...)`
- Минимум два адаптера:
  - in-memory adapter для unit-тестов;
  - file/jsonl adapter для ручного запуска без внешних сервисов.
- CLI или скрипт ручной проверки:
  - добавить факт;
  - найти факт по тексту/тегу;
  - создать связь;
  - вывести последние факты.
- README/документация запуска.

## Границы

### Делать

- Хранить только факты и связи.
- Явно отделить `fact.content` от любых выводов.
- Добавить валидацию, запрещающую пустой content/source.
- Сохранить расширяемость под embeddings/graph memory, но не требовать их.

### Не делать

- Не добавлять эмоциональную окраску.
- Не выводить привычки, принципы, навыки или идентичность.
- Не завязывать тесты на реальный mem0/API key.

## Интерфейс для соседних пакетов

Остальные пакеты должны иметь возможность передать dependency вида:

```python
memory: FactualMemory
```

и использовать его как черный ящик. Если язык проекта будет не Python, сохранить тот же смысл контракта в локальном стиле проекта.

## Проверки

- Unit-тесты на CRUD фактов.
- Unit-тесты на поиск по тегам и простому тексту.
- Unit-тесты на relations.
- Тест, что adapter не мутирует записанные факты при поиске.
- Команда ручного запуска из README.

## Definition of Done

- Пакет запускается без внешних сервисов.
- Все тесты проходят.
- В документации есть пример входа/выхода.
- Код не содержит ретроспективной интерпретации фактов.

## Готовый промпт для агента

Реализуй пакет Factual Memory Adapter по этому документу. Работай только с минимальным factual storage: факты, теги, связи, поиск, recent list. Не реализуй Experience Store, Identity Store или Reflection Engine. Итог должен запускаться локально без mem0 через in-memory/file adapter и иметь тесты.
