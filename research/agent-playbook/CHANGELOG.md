# Changelog

All notable changes to Agent Playbook will be documented in this file.

## [0.1.0] - 2026-04-30

### Added

**Core Patterns**:
- Pattern 01: Agent Rules Structure — Hierarchical, machine-readable documentation
- Pattern 02: Issue-to-PR Workflow — Asynchronous agent task management via GitHub
- Pattern 03: Strategic Review — Human oversight for system-level coherence
- Pattern 04: Environment Isolation — Safe parallel agent development
- Pattern 05: Context Handoff — Transferring knowledge between agents and sessions

**Guides**:
- Definition of Done for AI Agents — Clear completion criteria and quality gates
- Common Agent Failure Modes — Recognition and prevention of typical issues

**Templates**:
- AGENTS.md — Repository context and environment setup
- DEVELOPMENT_STANDARD.md — Project-specific terminology and patterns
- Pull Request Template — Structured change documentation
- Issue Templates:
  - Feature Request — New functionality proposals
  - Bug Report — Defect reporting with reproduction steps
  - Work Package — Large feature breakdown for parallel development

**Documentation**:
- README with pattern catalog and quick navigation
- Quick Start Guide for 15-minute setup
- LICENSE (MIT)

### Status

This is the initial release capturing patterns from real-world multi-agent development experience. All patterns are marked as "Stable" based on battle-testing in production projects.

### Origins

These patterns emerged from managing multiple autonomous AI agents working in parallel on a complex 7-component architecture project, coordinated entirely through GitHub workflows without direct agent access.
