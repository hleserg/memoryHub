# Установка Atman — пошаговая инструкция

> Читай внимательно, делай по порядку, не пропускай шаги.
> Если что-то пошло не так — не паникуй, читай раздел "Проблемы".

---

## Что у тебя должно быть перед началом

- Windows 10/11 с правами администратора
- RTX 5070 Ti с установленными драйверами NVIDIA
- Свободный NVMe диск
- Интернет (будем скачивать ~15 GB)

---

## Часть 1 — Подготовка Windows (делается один раз)

### Шаг 1.1 — Включить WSL2

Открой **PowerShell от администратора** (правой кнопкой на кнопке Пуск → Windows PowerShell (Администратор)) и выполни:

```powershell
wsl --install
```

Если WSL уже установлен, эта команда ничего не сломает. После — **перезагрузи компьютер**.

После перезагрузки откроется окно Ubuntu — создай имя пользователя и пароль (запомни их, они понадобятся).

---

### Шаг 1.2 — Проверить что WSL2 работает

В PowerShell:

```powershell
wsl --list --verbose
```

Должно показать что-то вроде:
```
  NAME      STATE           VERSION
* Ubuntu    Running         2
```

Важно чтобы VERSION был **2**, не 1. Если 1 — выполни:

```powershell
wsl --set-version Ubuntu 2
```

---

### Шаг 1.3 — Перенести WSL на NVMe диск

> Это нужно чтобы всё работало быстро и не забивало системный диск.

Узнай букву своего NVMe диска (например `D:` или `E:`). Открой Проводник — там видно диски.

В **PowerShell от администратора** замени `D:` на свою букву:

```powershell
# Создаём папку на NVMe
New-Item -ItemType Directory -Path "D:\WSL" -Force

# Экспортируем текущий Ubuntu (займёт пару минут)
wsl --export Ubuntu "C:\Users\%USERNAME%\ubuntu-backup.tar"

# Удаляем старый Ubuntu
wsl --unregister Ubuntu

# Импортируем на NVMe
wsl --import Ubuntu "D:\WSL\Ubuntu" "C:\Users\%USERNAME%\ubuntu-backup.tar"

# Устанавливаем дистрибутив по умолчанию
wsl --set-default Ubuntu

# Удаляем временный файл
Remove-Item "C:\Users\%USERNAME%\ubuntu-backup.tar"
```

---

### Шаг 1.4 — Настроить WSL (память и производительность)

В PowerShell создай файл настроек:

```powershell
notepad "$env:USERPROFILE\.wslconfig"
```

Вставь в открывшийся блокнот:

```ini
[wsl2]
memory=28GB
swap=8GB
processors=8
localhostForwarding=true
networkingMode=mirrored

[experimental]
autoMemoryReclaim=dropcache
sparseVhd=true
```

Сохрани файл (Ctrl+S) и закрой.

Перезапусти WSL:

```powershell
wsl --shutdown
```

---

### Шаг 1.5 — Исправить проблему с сетью WSL (важно!)

В PowerShell:

```powershell
# Открываем настройки сетевых адаптеров
ncpa.cpl
```

В открывшемся окне:
1. Найди адаптер **vEthernet (WSL)**
2. Правый клик → Свойства → Настроить → вкладка Дополнительно
3. Найди **Large Send Offload Version 2 (IPv4)** → поставь **Отключено**
4. Найди **Large Send Offload Version 2 (IPv6)** → поставь **Отключено**
5. ОК

## Часть 2 — Подготовка Ubuntu (внутри WSL)

Открой Ubuntu (найди в меню Пуск или в PowerShell напиши `wsl`).

### Шаг 2.1 — Обновить систему

```bash
sudo apt-get update && sudo apt-get upgrade -y
```

Введи пароль который создавал при установке. Подожди пока обновится.

---

### Шаг 2.2 — Установить uv

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc
```

Проверь:

```bash
uv --version
```

Должно показать версию.

---

### Шаг 2.3 — Проверить GPU из WSL

```bash
nvidia-smi
```

Должна показаться таблица с RTX 5070 Ti. Если видишь — всё хорошо, идём дальше.

Если пишет "command not found" — проблема с драйверами. Обнови NVIDIA драйверы на Windows.

---

### Шаг 2.4 — Смонтировать NVMe внутри WSL

Замени `/dev/sdb` на твой диск. Узнать его:

```bash
lsblk
```

Ищи диск размером как твой NVMe (например 500G или 1T), который не `/` (не системный).

```bash
# Заменяй /dev/sdb на свой диск!
sudo mkdir -p /mnt/nvme

# Если диск пустой (новый) — форматируем:
sudo mkfs.ext4 /dev/sdb

# Монтируем:
sudo mount /dev/sdb /mnt/nvme

# Добавляем чтобы монтировался автоматически при старте:
echo '/dev/sdb /mnt/nvme ext4 defaults 0 2' | sudo tee -a /etc/fstab

# Создаём папку для данных:
sudo mkdir -p /mnt/nvme/atman
sudo chown -R $USER:$USER /mnt/nvme/atman
```

Проверь что диск смонтирован:

```bash
df -h /mnt/nvme
```

Должно показать размер диска.

---

### Шаг 2.5 — Клонировать репозиторий Atman

```bash
cd /mnt/nvme/atman
git clone https://github.com/hleserg/atman.git
cd atman
```

## Часть 3 — Запуск скрипта установки

### Шаг 3.1 — Скопировать скрипт в репозиторий

Скрипт `atman-setup.sh` положи в папку `/mnt/nvme/atman/atman/`. Или скачай:

```bash
# Если скрипт лежит рядом с инструкцией — скопируй его в репозиторий
cp /путь/к/atman-setup.sh /mnt/nvme/atman/atman/atman-setup.sh
```

---

### Шаг 3.2 — Запустить установку

```bash
cd /mnt/nvme/atman/atman
bash atman-setup.sh
```

**Скрипт работает полностью автоматически.** Просто смотри на экран.

Что он делает по порядку:
1. Проверяет окружение, устанавливает недостающее
2. Находит NVMe диск, создаёт структуру папок
3. Генерирует пароли и ключи (сохраняет в `~/.atman/.secrets`)
4. Устанавливает и настраивает Ollama с Flash Attention
5. Скачивает `bge-m3` (~1 GB)
6. Скачивает `qwen3.5:9b` (~5.5 GB) — **это займёт время, не закрывай терминал**
7. Запускает PostgreSQL с pgvector в Docker
8. Создаёт всю схему БД (таблицы, индексы, политики безопасности)
9. Запускает Qdrant в Docker, создаёт коллекции
10. Устанавливает Python зависимости
11. Прогоняет smoke test

В конце увидишь зелёный блок "Atman Memory Stack успешно развёрнут!"

## Часть 4 — Проверка после установки

### Шаг 4.1 — Проверить что всё запущено

```bash
# Смотрим на контейнеры
docker ps

# Должны быть две строки:
# atman-postgres   Up ...
# atman-qdrant     Up ...
```

```bash
# Проверяем Ollama
ollama list

# Должны быть две модели:
# qwen3.5:9b
# bge-m3
```

---

### Шаг 4.2 — Проверить GPU

```bash
OLLAMA_DEBUG=1 ollama run qwen3.5:9b "привет" 2>&1 | grep "inference compute"
```

Смотришь на слово `library=`:
- `library=CUDA` → GPU работает, отлично
- `library=cpu` → GPU не определена (см. раздел Проблемы ниже)

---

### Шаг 4.3 — Проверить PostgreSQL

```bash
docker exec atman-postgres psql -U atman -d atman \
  -c "SELECT extname, extversion FROM pg_extension WHERE extname IN ('vector','pg_trgm');"
```

Должно вывести pgvector и pg_trgm с версиями.

```bash
# Проверить что роль приложения создана
docker exec atman-postgres psql -U atman -d atman \
  -c "SELECT rolname, rolsuper, rolcanlogin FROM pg_roles WHERE rolname = 'atman_app';"
```

Должно вывести `atman_app | f | t` (не суперпользователь, может логиниться).

---

### Шаг 4.4 — Проверить базовые тесты Atman

```bash
cd /mnt/nvme/atman/atman
source .venv/bin/activate
pytest tests/ -v --tb=short
```

Тесты без LLM должны пройти все зелёными.

---

### Шаг 4.5 — Запустить демо

```bash
make demo-session-fast
```

Должен отработать без ошибок.

## Секреты и роли PostgreSQL

Скрипт `gen-secrets.sh` генерирует два набора реквизитов:

| Переменная | Кто | Зачем |
|---|---|---|
| `ATMAN_ADMIN_DATABASE_URL` | `atman` (суперпользователь) | Только для миграций |
| `DATABASE_URL` | `atman_app` (не суперпользователь) | Приложение, RLS работает |

Оба URL хранятся в `~/.atman/.secrets` (chmod 600) и копируются в `.env` проекта. **Не коммитить в git.**

Если нужно подключиться к базе вручную:
```bash
# Приложение (RLS изолирует агентов)
docker exec -it atman-postgres psql "$DATABASE_URL"

# Администрирование (суперпользователь, RLS не применяется)
docker exec -it atman-postgres psql "$ATMAN_ADMIN_DATABASE_URL"
```

## Управление системой

### Запуск (если перезагружал компьютер)

```bash
# В WSL:
cd /mnt/nvme/atman/atman
docker compose up -d   # запускает PostgreSQL и Qdrant
# Ollama запускается автоматически как системный сервис
```

### Остановка

```bash
docker compose stop
sudo systemctl stop ollama
```

### Посмотреть логи

```bash
docker logs atman-postgres    # логи PostgreSQL
docker logs atman-qdrant      # логи Qdrant
journalctl -u ollama -f       # логи Ollama в реальном времени
```

### Бэкап БД

```bash
docker exec atman-postgres pg_dump -U atman atman \
  | gzip > /mnt/nvme/atman/backups/atman_$(date +%Y%m%d_%H%M%S).sql.gz
echo "Бэкап готов"
```

## Проблемы и решения

### GPU не определяется (library=cpu)

RTX 5070 Ti (Blackwell, CC 12.0) — новая архитектура, иногда требует свежей Ollama.

```bash
# Обновить Ollama
curl -fsSL https://ollama.com/install.sh | sh
sudo systemctl restart ollama
```

Если не помогло — попробовать Vulkan backend:

```bash
sudo tee /etc/systemd/system/ollama.service.d/override.conf > /dev/null << 'EOF'
[Service]
Environment="OLLAMA_MODELS=/mnt/nvme/atman/ollama/models"
Environment="OLLAMA_FLASH_ATTENTION=1"
Environment="OLLAMA_VULKAN=1"
Environment="OLLAMA_KEEP_ALIVE=5m"
EOF
sudo systemctl daemon-reload && sudo systemctl restart ollama
```

### Docker требует sudo

```bash
sudo usermod -aG docker $USER
# Закрой и открой терминал WSL заново
```

### WSL не видит NVMe диск

В PowerShell (Windows):

```powershell
wsl --mount \\.\PHYSICALDRIVE1 --bare
```

Номер диска (`PHYSICALDRIVE1`) узнай в Диспетчере устройств или через:

```powershell
Get-Disk | Select Number, FriendlyName, Size
```

### Мало памяти при скачивании модели

```bash
# Смотрим сколько свободно
free -h

# Если мало — увеличиваем файл .wslconfig на Windows:
# memory=28GB  (или сколько есть минус 4GB для Windows)
# После изменения: wsl --shutdown в PowerShell
```

### PostgreSQL не запускается

```bash
docker logs atman-postgres
# Читаем что написано, ищем ERROR

# Если проблема с данными:
docker rm atman-postgres
docker volume rm atman-postgres-data
# И запускаем скрипт снова
```

### Скрипт упал на каком-то шаге

Скрипт идемпотентный — можно запускать повторно. Уже сделанные шаги просто пропустит.

```bash
bash atman-setup.sh
```
